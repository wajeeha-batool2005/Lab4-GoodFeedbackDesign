# Lab 04 - HCI Principles Demo (Colored for Visibility)

## Overview
This project demonstrates core **Human-Computer Interaction (HCI) principles** using a simple interactive button. The lab highlights **affordance, mapping, feedback, constraints, and signifiers** through color-coded visual cues.

---

## Features
- **AFFORDANCE**: Button has a blue border to indicate it is clickable.
- **MAPPING**: Status box with green border shows results and communicates the system state.
- **FEEDBACK**: Yellow background animation on button click indicates the action has been registered.
- **CONSTRAINT**: Button disables after reaching a click limit (2 clicks), with a red border.
- **SIGNIFIER**: Purple background and opacity indicate the button is no longer usable.

---

## How to Run
1. Clone or download the repository.
2. Open `lab4.html` or `index.html` in any web browser.
3. Click the **Process Data** button to see the HCI principles in action:
   - Observe color changes for feedback.
   - Note how constraints and signifiers indicate the click limit.

---

## Notes on HCI Principles
- **Affordance**: Button visually suggests it can be clicked.
- **Mapping**: Status box visually maps action to result.
- **Feedback**: Immediate visual response on click.
- **Constraint**: Limits user actions (max 2 clicks) to prevent errors.
- **Signifier**: Disabled button shows visual cues to communicate unavailability.
